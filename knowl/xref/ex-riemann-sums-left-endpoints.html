Example 4.2.5.

Consider the function \(r(t)=t^2\) on \([0,2]\)
🔗
(a) Calculate the left Riemann sum with \(n=4\) subintervals.
🔗
(b) Is you answer from (a) an underestimate or overestimate of the total change, and why?
🔗
Solution.
(a) The width of each subinterval is
🔗
\begin{equation*} \Delta t = \dfrac{\text{total width of interval}}{\text{number of subintervals}} = \dfrac{2-0}{4} = \dfrac12 \end{equation*}
If we split up the interval into four subintervals, we get \([0,\tfrac12]\text{,}\) \([\tfrac12,1]\text{,}\) \([1,\tfrac32]\text{,}\) and \([\tfrac32,2]\text{.}\) The left endpoints are \(0, \tfrac12, 1, \tfrac32\text{.}\) So the left Riemann sum is
🔗
\begin{equation*} \begin{array}{rcl} \amp \amp r(0)\Delta t + r(\tfrac12)\Delta t + r(1)\Delta t + r(\tfrac32)\Delta t \\ \amp = \amp (0^2)(\tfrac12) + (\tfrac12)^2(\tfrac12) + (1^2)(\tfrac12) + (\tfrac32)^2(\tfrac12) = \dfrac{7}{4}. \end{array} \end{equation*}
(b) Since \(r(t)=t^2\) is increasing on \([0,2]\text{,}\) the left hand endpoints will give us smallest values of \(r(t)\) on each piece (subinterval). So the left Riemann sum will be an underestimate of the total change. You can verify that this will be equivalent to the lower Riemann sum in this case, since the function is increasing.
